## Post-release source-role write validation

Read-only Home Assistant state and config-entry inspection on 2026-07-01
identified a safe candidate for the reversible source-role write proof:

- Zero Net Export config entry:
  `01KWC2HX12V4P0Q82A0WM69EHV`
- Current optional `battery_soc_entity` value: unset
- Candidate entity:
  `sensor.x1_p6k_us_s_state_of_charge`
- Candidate state evidence: `75`, unit `%`, `device_class=battery`,
  `state_class=measurement`

After approval, the supported `zero_net_export.update_source_roles` service
was called through the Home Assistant API. No direct Home Assistant storage
write was used.

Evidence:

- Before write:
  `sensor.zero_net_export_battery_state_of_charge_status=missing`,
  binding `null`, reading `unknown`.
- Service set call:
  `entry_id=01KWC2HX12V4P0Q82A0WM69EHV`,
  `battery_soc_entity=sensor.x1_p6k_us_s_state_of_charge`; Home Assistant
  service response `[]`.
- After set:
  `sensor.zero_net_export_battery_state_of_charge_status=ok`,
  binding `sensor.x1_p6k_us_s_state_of_charge`, reading `75.0`, unit `%`,
  `device_class=battery`, `state_class=measurement`, issue counts
  `error=0`, `warning=0`, `info=0`.
- Service restore call:
  same entry with `battery_soc_entity=""`; Home Assistant service response
  `[]`.
- After restore:
  `sensor.zero_net_export_battery_state_of_charge_status=missing`,
  binding label `Not configured`, reading `unknown`, issue counts
  `error=0`, `warning=0`, `info=0`.
- Read-only storage check after restore confirms
  `battery_soc_entity=""`, which is the supported service-path unset value.
- Installed version remained `sensor.zero_net_export_installed_version=0.2.3`;
  HACS update metadata remained installed/latest `v0.2.3`.
- Artifacts:
  `tmp/zne-source-write-validation-summary.json`,
  `tmp/zne-source-write-set-response.json`,
  `tmp/zne-source-write-restore-response.json`,
  `tmp/ha-states-v0.2.3-source-write-before.json`,

  `tmp/ha-states-v0.2.3-source-write-restored.json`,

The existing `battery_discharge_power_entity` is not a safe disposable

the historical log evidence is tracked separately as ZNE-594.
