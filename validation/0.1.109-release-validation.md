# Zero Net Export 0.1.109 release validation

Date: 2026-05-06
Release scope: ZNE-592 corrective release for misleading managed-load edit/remove button rows introduced in `0.1.108`.
Repo commit validated: `50e6a44`
Manifest version: `0.1.109`

## Feasibility gate

Fresh Home Assistant frontend feasibility was completed before the corrective implementation.

Evidence: `validation/zne-592-native-clickthrough-feasibility.md`.

Findings:

- Supported: managed-load child devices can expose a Home Assistant native `configuration_url` to `/config/integrations/integration/zero_net_export`.
- Supported: the integration page exposes native Configure/options flow affordances where the Managed Devices workspace lives.
- Unsupported: Home Assistant button entity rows cannot directly open a selected managed-load edit step or native remove confirmation on click.
- Correction: remove the misleading edit/remove button rows and use the supported native configuration URL/integration Configure route instead. The backend `zero_net_export.remove_managed_device` service remains available for advanced/manual/scripted removal.

## Release readiness decision

Repo/live validation passes for the corrective scope of `0.1.109`.

ZNE-592 is validated fixed under the supported Home Assistant-native path: misleading direct edit/remove button rows are gone, and the visible supported path now routes operators to the Zero Net Export integration page/Configure flow rather than a generic button more-info dialog.

Remaining known limitation: Home Assistant still does not provide a supported direct native selected-managed-load remove confirmation from the device-page entity row. That exact direct row workflow remains unsupported unless upstream Home Assistant adds an extension point or Riley approves custom/experimental frontend work.
