# Zero Net Export 0.2.1 release validation

Date: 2026-06-30
Status: release preparation in progress; GitHub/HACS-only path approved

## Scope

Release `0.2.1` validates the second Home Assistant application workflow slice
through the approved release-management path. All live Home Assistant changes
must flow through GitHub release publication and HACS install/update; no direct
Home Assistant custom-component file backend deployment is approved.

## Candidate contents

- In-app navigation for Overview, Sources, Managed Devices, Controls, Runtime,
  Diagnostics, and Settings.
- App-side managed-device enable/disable actions scoped by config entry and
  managed-device key.
- Strong-confirmation managed-device removal using `REMOVE FROM ZNE`, preserving
  the original/source Home Assistant device/entity.
- App-side edits for existing controller enabled, mode, target export, deadband,
  and battery reserve control paths.

## Required release path

- Commit intended `0.2.1` files.
- Push `main`.
- Create/push tag `v0.2.1`.
- Publish GitHub Release `v0.2.1`.
- Verify GitHub latest release and tag manifest report `0.2.1`.
- Refresh Zero Net Export in HACS with **Update information**.
- Install/update to `0.2.1` through HACS.
- Restart Home Assistant.
- Compare installed fingerprint against the `0.2.1` repo artifact.
- Review project-specific Home Assistant logs.

## Required live validation

- Desktop browser proof for the app workflow shell.
- Narrow/mobile browser proof for the app workflow shell.
- Sidebar/app route proof for the installed `0.2.1` package.
- Write-path proof for one safe control edit.
- Strong-confirmation proof for managed-device remove using a disposable managed
  load.
- Proof the original/source Home Assistant device/entity remains present after
  the managed record is removed.
