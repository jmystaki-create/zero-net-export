# v0.3.1 Fix - Manual Update Instructions

**Date**: 2026-07-07  
**Status**: READY FOR MANUAL UPDATE

## What Was Fixed

The v0.3.1 release had a manifest.json version mismatch:
- GitHub tag v0.3.1 pointed to a commit with `"version": "0.3.0"` in manifest.json
- Home Assistant/HACS reported version 0.3.0 even though the release was v0.3.1

**Fix applied**:
- Updated manifest.json to `"version": "0.3.1"`
- Updated the v0.3.1 tag on GitHub to point to the corrected commit (8c3fa26)
- All code changes are pushed to GitHub

## Current State

- **GitHub**: v0.3.1 tag now points to the corrected commit with manifest version 0.3.1
- **Home Assistant**: Still showing version 0.3.0 because HACS hasn't re-downloaded the updated files
- **HACS Cache**: HACS has cached the old v0.3.1 files and hasn't automatically refreshed

## Manual Update Required

Since HACS doesn't have an API to force a repository refresh, you need to manually trigger the update:

### Option 1: HACS UI (Recommended)

1. Open Home Assistant
2. Go to **HACS** → **Integrations**
3. Find **Zero Net Export**
4. Click the three-dot menu (⋮) in the top-right
5. Select **Download** or **Redownload** (if available)
6. If not available, click **Show available updates** or **Check for updates**
7. If an update is shown, click **Download**
8. Restart Home Assistant after the download completes

### Option 2: Force HACS Refresh

1. Open Home Assistant
2. Go to **HACS** → **Integrations**
3. Click the three-dot menu (⋮) in the top-right
4. Select **Custom repositories**
5. Remove **Zero Net Export** from the list
6. Click **Add repository** and re-add `jmystaki-create/zero-net-export`
7. Install the integration again
8. Restart Home Assistant

### Option 3: Wait for Automatic Refresh

HACS automatically refreshes repository data every few hours. If you wait 2-4 hours, HACS should detect the updated tag and show an update available.

## Verification After Update

After completing the manual update and restarting Home Assistant:

1. Check `sensor.zero_net_export_installed_version` → should show `0.3.1`
2. Check `update.zero_net_export_update` → should show latest version as `v0.3.1`
3. Open the Zero Net Export sidebar app
4. Click the "Diagnostics" tab
5. Verify all sections render correctly:
   - Release information (should show v0.3.1)
   - System Health summary
   - Log Buffer (last 10 entries)
   - Reconciliation Trend (last 7 entries)
   - Download/Copy/Repair buttons
