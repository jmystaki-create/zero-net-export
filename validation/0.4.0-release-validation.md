## Log Review

Targeted Home Assistant log review after the v0.4.0 install found:

- Historical pre-v0.4.0 diagnostics `NameError: name '_LOGGER' is not defined` from earlier export attempts; this was already fixed before v0.4.0.
- Standard Home Assistant custom-integration warnings.
- No observed exception from the explicit v0.4.0 `export_diagnostics`, `repair_issue`, `pause_executor`, or `resume_executor` service calls in the reviewed window.
- Recorder warnings beginning around `2026-07-07 20:24:31` where many Zero Net Export entities exceeded Home Assistant's 16 KB stored-attributes limit.

## Follow-Up Risk

The recorder warnings are a new high-priority follow-up. They indicate ZNE is attaching too much diagnostic/runtime detail to entity attributes; Home Assistant suppresses storage for those attributes and warns about database performance.

Recommended next work:

- Create/fix a focused bug to trim or move large diagnostic/action-history/source detail out of recorder-backed entity attributes.
- Keep concise entity attributes for dashboards and expose bulky diagnostics through diagnostics export/app API instead.

## Result

Milestone 7 is released and live validated for the available API/static/service scope on installed `v0.4.0`. Browser visual proof remains pending until the managed browser environment is repaired.
